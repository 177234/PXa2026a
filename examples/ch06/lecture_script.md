# 课堂演示动线讲稿 (约 8 分钟正片 + 2 分钟缓冲)

**演示对象**：Baker, Callaway, Cunningham, Goodman-Bacon & Sant'Anna (2026),《DiD 实践者指南》,*JEL* 64(2): 498–557。复现包：<https://github.com/pedrohcgs/JEL-DiD>
**一句话卖点**：这篇论文的作者就是你们课本里 DID 方法的提出者们；今天我们让 AI Agent 从他们的复现包出发，把论文里的表和图原样跑出来。

---

## 课前准备清单 (缺一不可)

1. 下载 Release 并解压；把 `run_demo.do` (或 `run_demo.R`) 复制到包根目录，改好路径。
2. **完整干跑一遍**：让软件包 (SSC 或 renv) 全部装好缓存，记录实际耗时。
3. 干跑后**清空 figures/ 与 tables/ 里新生成的文件** (或整包重新解压一份)，保证现场「从零跑出」；留一份干跑成功的输出作保险副本。
4. Agent 里贴好「分步演示版」提示词 (`demo_prompts.md` 的 A 版)，路径已替换。
5. 打开论文 PDF，预先停在 2×2 主结果表与事件研究图那两页，备第 5 幕对照用。

## 演示动线

**第 0 幕 | 开场 (30 秒)**
台词：「复现是实证研究的成人礼。我们今天不自己写代码，而是当『审稿人 + 指挥官』，让 Agent 去干活。数据是美国 3,143 个县 2010–2019 年的面板：各州先后扩张了医保 (Medicaid)，我们看它对成年人死亡率的影响。」

**第 1 幕 | Agent 读 README、复述复现计划 (1 分钟)**
讲点：一个规范复现包的「四件套」——README、data/、scripts/、输出目录。让学生看 Agent 如何从 README 推出运行顺序。

**第 2 幕 | 清点数据 (1 分钟)**
讲点：复现第一步永远是**核对数据而不是跑代码**。让 Agent 报出观测数、变量清单，与 README 描述对上。

**第 3 幕 | 跑 2×2 DID (2 分钟) ★全场核心**
在 Agent 跑的同时，你在黑板/幻灯写：
`Y_it = α + β1·Treat_i + β2·Post_t + β3·(Treat_i × Post_t) + ε`
台词：「注意，所谓双重差分，就是你们前面学的**两个虚拟变量的交叉项**——β3 就是政策效应。」表一出来，指着交叉项那行念系数。

**第 4 幕 | 跑事件研究、出图 (1.5 分钟)**
台词：「把 Post 换成一串年份虚拟变量，和处理组虚拟变量逐个交乘，就得到这张图——每个点是一个交叉项系数。处理前的点都贴着 0 走，这就是平行趋势的可视化。」

**第 5 幕 | 与论文原表并排对照 (1.5 分钟) ★最有说服力的一幕**
让 Agent 把跑出的表和论文 PDF 的表并排，逐项报「一致 / 舍入差异 / 不一致」。台词：「数字对上了——这就叫可复现。你们的期末作业，我也会用同样的标准来跑。」

**第 6 幕 | 收尾与彩蛋 (30 秒)**
「包里还有 4、5 号脚本——多期错峰 DID 和敏感性分析，那是高阶课的内容，留个念想。课后作业：换另一种语言 (R↔Stata) 把今天的结果再复现一遍，或者改一个控制变量看结果稳不稳。」

## 风险预案

- **网络慢 / 装包卡住** → 因为课前已缓存，现场不应再下载；若意外触发，切到保险副本输出继续讲第 5 幕。
- **路径报错** → 让 Agent 自己诊断修复——这本身就是演示点，不必慌。
- **某张表没出来** → 只讲已生成的部分，并当场让 Agent 定位原因，顺势讲「复现失败也是有价值的发现」。
- **彻底翻车** → 打开保险副本的表图 + 论文 PDF，人工完成第 3–5 幕的讲解，演示价值不减半。
