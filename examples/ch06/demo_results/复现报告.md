# 第六讲现场演示 · 预跑复现报告 (标准答案)

> 这是老师课前 (T7 彩排) 在本机把 JEL-DiD 复现包**实际跑通一遍**后冻结的结果，作为课堂演示的「标准答案」。演示时老师让 Agent 现场跑；你 (学员) 课后自己跑完，就用这个文件夹里的表和图来对比——数字对得上，就说明你复现成功。

## 一、跑了什么

- **复现对象**：Baker, Callaway, Cunningham, Goodman-Bacon & Sant'Anna (2026). Difference-in-Differences Designs: A Practitioner's Guide. *JEL*, 64(2), 498–557. 复现包：<https://github.com/pedrohcgs/JEL-DiD> (MIT 许可)。
- **数据**：`county_mortality_data.csv`，美国县级面板 (约 3,100 个县 × 2009–2020)，看各州先后扩张医保 (Medicaid) 对成年人 (20–64 岁) 死亡率的影响。
- **运行范围**：只跑初级班范围内的 `0`→`1`→`2`→`3` 号 Stata 脚本 (数据构建、采纳时点表、2×2 DID、事件研究)，**跳过** `4_GxT` (多期错峰估计) 与 `5_honestdid` (敏感性分析)——这两块是现代 DID 内容，留给高级班。
- **环境**：Stata 19 MP (Windows)；总耗时约 5 分钟 (含首次联网装包)，纯分析约 2–3 分钟。

## 二、验证结论 (关键)

用 `git diff` 把本机重跑的输出和作者随包提交的参考输出逐一比对：

| 输出 | 比对结果 |
|---|---|
| **Table 1–6** (`_stata.tex`) | **逐字节完全一致**——精确复现 |
| **Table 7** (`_stata.tex`) | **点估计完全一致**；只有标准误在小数点后第二位有微小差异 (如 4.68 vs 4.67) |
| **Figure 1–4** (`_stata.pdf`) | 重新生成，内容与参考图一致 (PDF 文件因内嵌时间戳，二进制层面必然不同，不代表结果不同) |

Table 7 那点标准误差异，**不是错误，而是一个绝好的教学点**：它用的是 Sant'Anna–Zhao (2020) 双重稳健估计量 (`drdid` 命令)，标准误靠**自助法 (bootstrap)** 算，每次运行会有极小的随机浮动。所以「点估计对得上、自助标准误末位有微差」正是**可复现的正常表现**——这也是为什么核对结果要分清「完全一致 / 数值层面一致 (仅舍入或自助差异) / 不一致」三档。

## 三、头条结果 (演示时指着念的那几个数)

- **简单 2×2 DID** (Table 2、Table 3)：医保扩张 × Post 的**交叉项系数**——人口加权约 **−2.6** (每十万成年人死亡数，marginally significant)，非加权约 **0**。这就是「双重差分 = 两个虚拟变量的交叉项」最直观的落地。
- **加控制变量后** (Table 5)：加权 Treat 系数约 **−2.56**，方向与量级稳定。
- **事件研究** (Figure 3、Figure 4)：处理前各期系数贴着 0 走 (平行趋势的可视化)，处理后向下——把 `Post` 换成一串年份虚拟变量、与处理组虚拟变量逐个交乘的结果。

一句话：**医保扩张与成年人死亡率的小幅下降相关**，加权后约每十万人少 2.6 例，勉强显著；这正是论文用来讲解 DID 全套工具的那个示例结论。

## 四、这个文件夹怎么用

```
demo_results/
├── 复现报告.md      ← 本文件 (标准答案说明)
├── tables/          ← table1–7_stata.tex，本机重跑冻结 (= 作者参考，Table 7 除自助 SE 外一致)
├── figures/         ← figure1–4_stata.pdf，2×2 DID 与事件研究图
└── run_log_full.txt ← 完整运行日志 (预录兜底证据)
```

- **课后自查**：你按 `../demo_prompts.md` 让 Agent 跑完后，把你的 `tables/table3_stata.tex` (2×2 DID 主表) 和这里的对照，核心交叉项系数应当一致；标准误若在末位有微小出入，属正常。
- **现场兜底**：万一课堂网络或 Agent 抽风，直接打开本文件夹的表和图，配合论文 PDF 完成讲解，演示价值不减。

## 五、一个复现常识 (顺带教学)

`run_log_full.txt` 里能看到：脚本 0 读的是包内已有的 `county_mortality_data.csv`，全程不需要联网取原始数据——**好的复现包会把最终分析数据一起打包，为断网留后路**。这也是判断一个复现包是否规范的一个信号。
