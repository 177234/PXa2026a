# ch02 示例数据说明(模拟数据)

> 本讲的脊柱数据。**全部为模拟生成**，并非任何真实数据库导出，仅用于教学。其结构参照真实的
> 上市公司-年度研究数据，并**刻意保留了实证工作中最常见的几类毛病**(缺失、离群、文字型数字、
> 键不一致、粒度不齐、分年导出)，供全章逐节演示。数据由 `00_make_ch02_data.do` 生成(可复现)；
> 学员也可换成自己的数据，按同一条流程练习。

## 研究情境(旅程的主人公)

研究情境：考察**制造业上市公司的研发投入是否随企业规模变化、并因行业而异**，需要一张「一家公司
一年一行」、变量口径清楚、可直接分析的**公司-年度面板**。可原始文件是分头导出的几份表，离「能
分析」还差整整一讲的距离。

数据规模：**约 42 家公司 × 2020–2022 三年 = 126 个公司-年度观测**。其中 6 家为「锚点公司」
(`000101`、`000102`、`600201`、`600202`、`000301`、`000302`)，手工设定、**承载全部教学毛病**
(缺失、离群、文本型百分比、前导零)；其余 36 家(`200001`–`200036`)由确定性公式生成、数据完整、
取值有真实散布，用于让分布与 EDA 图形有说服力。三个行业：`C39` 电子、`C27` 医药、`C36` 汽车。

## 原始文件(旅程的起点)

| 文件 | 是什么 | 观察单位 | 刻意保留的毛病(教学点) |
|---|---|---|---|
| `firm_finance_2021.dta`、`firm_finance_2022.dta` | 分两年导出的公司财务表，需先 append | 公司-年度 | `rd` 研发支出部分缺失(空值或写成 -99)；某公司 `sales` 误多打一个 0(离群)；`lev` 资产负债率存成文字如 `"45.3%"`；`stkcd` 含前导零如 `"000063"` |
| `industry_lookup.dta` | 行业代码对照表，用于 m:1 合并 | 行业(横截面) | 个别 `indcd` 带空格或大小写不一，清洗后才能匹配 |
| `mktcap_monthly.dta` | 月度市值表，聚合到年度后再合并 | 公司-月 | 一家公司一年有 12 行，直接按 `stkcd` 合并到年度表会**爆行**，须先聚合；且此表 `stkcd` 存成了**数值型**(前导零丢失)，与财务表的**字符型** `stkcd` 键不一致 |

主要变量(原始态)：`stkcd`(证券代码)、`year`(年度)、`sales`(营业收入，万元)、
`rd`(研发支出，万元)、`at`(总资产，万元)、`lev`(资产负债率)、`indcd`(行业代码，如
`C39` 计算机通信电子、`C27` 医药、`C36` 汽车)、`ym`(年月，如 `202103`)、`mktcap`(月末市值)。

## 目标(旅程的终点)：`firm_year_clean.dta`

- **观察单位**：公司-年度；**主键**：`stkcd + year`(唯一)。
- **变量字典(节选)**：

| 变量 | 含义 | 口径 / 构造 |
|---|---|---|
| `stkcd` | 证券代码 | `str6`，保留前导零 |
| `year` | 年度 | |
| `indname` | 行业名称 | 由 `indcd` 合并 `industry_lookup` 得到 |
| `sales` | 营业收入(万元) | 离群值已核查修正 |
| `rd_ratio` | 研发强度 | `rd / sales`，缺失与离群处理后、已缩尾 |
| `size` | 企业规模 | `ln(at)` |
| `lev` | 资产负债率 | 由文字转数值，取 0–1 |
| `mktcap_yr` | 年度市值 | 由月度市值聚合(年末或年均) |

- **伴随产物**：样本筛选日志、变量字典、合并匹配率与样本流失表(见 §2.7)。

## 与主案例 Lane (2025) 的呼应

Lane 的分析数据是**行业-年度**面板，合并键是**行业代码**，处理变量由「目标行业 × 政策后年份」
构造——与小林的「公司-年度、按行业码合并、按规则构造变量」在结构上**同构**。§2.8 会把本讲这套
做法迁移到 Lane 的真实(已清洗)数据上，做轻量复现与验证，让学员看到：同一套判断与技能，既能
整理自己的小数据，也能读懂顶刊的真数据。
