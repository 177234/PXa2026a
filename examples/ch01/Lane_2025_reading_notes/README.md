> 本文件夹是第 1 讲的配套示例：**如何借助 AI，用「任务切割」的方式快速读懂一篇顶刊论文**。
>
> 主案例：Lane, N. (2025). *Manufacturing Revolutions: Industrial Policy and Industrialization in South Korea.* **QJE**, 140(3), 1683–1741.
> 论文链接：<https://doi.org/10.1093/qje/qjaf025>｜复现包：<https://doi.org/10.7910/DVN/VJECHN>

## 为什么要「切割任务」

面对一篇 40 多页的顶刊论文，初学者最容易犯的错误，是**一上来就把整篇 PDF 丢给 AI，让它写一份大而全的完整读书笔记**。这样做有三个问题：

- AI 要在一次回答里兼顾背景、方法、数据、结论、复现，往往每一项都浅尝辄止；
- 输出很长，但你读完还是抓不住重点，也难以核对；
- 你无法带着**具体问题**追问，学习变成了被动接收。

更好的做法是**把「读一篇论文」拆成几个角度，每个角度让 AI 单独生成一份短笔记**。每份只回答一类问题，你也更容易判断它写得对不对。这样，**花一个上午读懂三四篇顶刊、判断哪些值得精读**，就成了现实。

## 四份笔记，四个角度

| 笔记 | 角度 | 回答什么 | 配套 skill |
|---|---|---|---|
| [`01_overview.qmd`](01_overview.qmd) | **整体介绍** | 研究背景、研究问题、研究思路、主要结论——最快了解全貌 | `01-paper-context` |
| [`02_methods.qmd`](02_methods.qmd) | **实证方法与识别** | 用了什么实证方法、识别策略是什么、怎么处理内生性（**只讲是什么、为什么，不推导公式**） | `02-paper-strategy` |
| [`03_replication.qmd`](03_replication.qmd) | **复现材料导览** | 复现包怎么组织、主程序在哪、数据分几层、能复现到什么程度 | `03-replication-navigator` |
| [`04_literature.qmd`](04_literature.qmd) | **迷你文献定位** | 这篇论文站在哪些文献之间、如何向前向后横向找到密切相关的高水平文献 | `01-paper-context` |

**使用建议**：先读 `01_overview.qmd` 判断这篇文章是否与你的研究相关；相关，再读 `02` 看方法能否迁移；想动手复现，再看 `03`；想顺藤摸瓜找文献，用 `04`。

每份笔记开头都给出了**生成它的提示词**和**建议调用的 skill**。两种用法任选：**只复制提示词**发给任意 AI 助手（网页版 ChatGPT、Claude/Claude Code、Codex、OpenCode 都行）即可，零门槛、无需配置；想让流程更稳定、可复用，再进一步把提示词固化成 skill 调用。skill 是什么、怎么装、怎么在不同 agent 里触发，见课程主页《[AI 工作流与 skills 上手](https://lianxhcn.github.io/PXa2026a/skills-guide.html)》。**不熟悉 skill 完全不影响本讲**——先用提示词起步即可。

## 关于难度

本课程是初级班。这几份笔记里，`02_methods.qmd` 会点到 DID（双重差分）、事件研究、双重稳健、三重差分等名词，但**只解释它们大致是什么、作者为什么用**，不做公式推导。真正的方法细节留给后续章节和高级班。遇到看不懂的名词，把它当作「知道有这么个工具」即可，不必卡住。

## 进阶参照：一位博士生的多角度精读成品

`_materials/ch01/Lane_2025_QJE_notes/`（课程素材）里，有一位博士生用 agent 对这篇论文生成的一整套精读笔记：framing / summary / methods / derivation / replication 五个角度，外加逐行代码注释和一份多 agent 协作工作流（`codex-workflow.md`）。

- 它是本方法的**高配版成品**，可以拿来**观摩**「用 AI 读顶刊能做到什么程度」；
- 其中 methods / derivation 两份含较多计量方程，属**进阶内容**，初级班同学看不懂很正常，点到为止；
- `codex-workflow.md` 展示的是「监督员 + 执行员」双 agent 长时间协作的工程化流程，属**高级可选**，不要求初学者复刻。

我们课上做的，是它的**简化版**——用几条提示词加 skill，在网页端就能跑起来的那一版。
